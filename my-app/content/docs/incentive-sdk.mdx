## Overview

`liquidlink-sdk` wraps the Move scoreboard package so any app can query point balances, run linear-time calculations, and read on-chain events from Sui or IOTA. It is built on top of `unimove-sdk` and exposes a thin, chain-aware client.

## Installation

```bash
npm install liquidlink-sdk
```

## Quick Example

```ts
import { Client } from 'liquidlink-sdk';

const sdk = new Client({ chain: 'iota', network: 'testnet' });

const REGULAR_SCOREBOARD_ID = '0x...';
const TARGET_ADDRESS = '0x...';

console.log(
  await sdk.getScoreInfo({
    scoreboardId: REGULAR_SCOREBOARD_ID,
    targetAddress: TARGET_ADDRESS,
  }),
);

console.log(
  await sdk.getComputedScore({
    scoreboardId: REGULAR_SCOREBOARD_ID,
    targetAddress: TARGET_ADDRESS,
    opts: { linear: false },
  }),
);

console.log('Available events', sdk.getAvailableEventTypes());
```

## Client API

### `new Client({ chain, network, rpcUrl? })`

Creates an SDK instance for a specific chain/network. `chain` is `"sui"` or `"iota"`, `network` is `"mainnet"` or `"testnet"` (default `"mainnet"`), and `rpcUrl` optionally overrides the default fullnode URL.

### `getClient()`

Returns the underlying chain client (e.g., Sui/IOTA SDK client) for direct RPC access when needed.

### `getConfig()`

Returns the pre-baked package metadata (package IDs, router addresses, etc.) for the selected chain/network.

### `getAvailableEventTypes()`

Lists fully-qualified Move event types emitted by the incentive system on the configured chain. Useful when filtering events.

### `getScoreInfo({ scoreboardId, targetAddress })`

Fetches the raw `score`, `scorePerDuration`, `duration`, and `updatedAt` stored for an address in a scoreboard. Returns zeros if the entry does not exist.

### `getComputedScore({ scoreboardId, targetAddress, opts? })`

Runs either a plain score lookup or a linear-time computation (when `opts.linear` is `true`) using the stored state. Returns `computedScore` along with the raw fields.

### `getEventsByAddress({ targetAddress, eventTypes?, scoreboardId?, scoreboardIds?, cursor?, limit?, order? })`

Reads events emitted by the scoreboard modules, filtered by sender address. Either `scoreboardId` or `scoreboardIds` must be provided. Optional filters:

- `eventTypes`: limit to specific event type strings.
- `cursor` / `limit` / `order`: pagination and sort options (`order` defaults to `"descending"`).

Returns `{ data, cursor, hasNextPage }` where `cursor` can be reused for pagination.
